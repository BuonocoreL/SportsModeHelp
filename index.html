SportsMode Trouble Shooting Process and Procedures

When a user reports an issue using the Report Issue feature from the SportsMode menu, or by sending an email to the 2016-Help@espn.com email address, a ticket is created in the 2016 Support project in Jira. Using the Report Issue feature, has the added benefit of including all of the relevant debug information from the user's application and is the preferred method for reporting issues.

Jira tickets will automatically get assigned to Statz’s team and the team will automatically receive notification from Jira of the new issue.

For every issue received from a user, contact the user advising them that their ticket has been received and is under review.

Likely Issues Encountered

  • Rundown VOD clip doesn’t play (user enters failed rundown item)
  • No rundown (panel empty, not between expected times)
  • Fantasy team not updating correctly (user enters faulty team/player/score)
  • Program information incorrect (user enters incorrect program/channel)
  • Live Rundown element failed to tune
  • Guide failed to tune (user enters failing program/channel)
  • Scoreboard Failed to Tune (user enters failed game)
  • Hear audio-only, no video
  • Not receiving likes or shares

Rundown VOD Clip Doesn't Play

  1. Attempt to play the same VOD on a mirror setup stack following the steps outlined in Step #4 then continue to Step #2. (Rundown text is in the ticket – user entered it.)
  2. Contact user and arrange support on the user's device.
  3. Confirm proper deployment stack.
  4. Attempt to play same VOD on the user's stack. (Rundown text is in the ticket - user entered it.)
    • If element has video available (white TV icon) but the “tune in” option is disabled, escalate/re-assign to DeRon Brown.
    • If the VOD plays:
      • If first time playing, resolve the ticket as “Cannot Reproduce”.
      • Otherwise, resolve the ticket as “Fixed”.
    • If the VOD plays, but you hear only audio, hard reset the Google TV (unplug) and return to Step #4
    • If the VOD does not play:
      • Check device network connection
        • If has connection (Ethernet/Wifi), continue.
        • If no connection, reconnect to network. Return to Step #4
      • Check HLS server connection
        • Open the Debug Console (press ‘d’ on the remote)
        • Check HLS Video Server Latency
          • If PASS (has millisecond value), continue.
          • If FAIL, escalate/re-assign to Matt Tretin.
      • Check known HLS video
        • Open the Debug Console (press ‘d’ on the remote)
        • Under Video, play the test HLS video stream
          • If it plays correctly, continue.
          • If it plays, but you hear audio-only with black video, hard reset the Google TV (unplug) and return to Step #4.4.3.
          • If it fails to play, escalate/re-assign to Matt Tretin.
      • Check the Rundown HLS video
        • Obtain the Rundown VOD URL from the debug report (match based on text in ticket).
        • Attempt to play the URL in QuickTime or VLC.
          • If it plays, escalate/re-assign to DeRon Brown.
          • If it fails to play, escalate/re-assign to Matt Tretin.

No Rundown (Panel Empty Not Between Expected Times)

  1. Check the availability of the rundown on a mirror setup stack. Follow the steps outlined in Step#4 then continue to Step#2.
  2. Contact the user and arrange support on user's device.
  3. Confirm proper deployment stack.
  4. Check the availability of the rundown on the user's stack.
    • If Rundown is available:
      • If first time testing, resolve ticket as “Cannot Reproduce”.
      • Otherwise, resolve the ticket as “Fixed”.
    • If Rundown is not available:
      • Check device network connection
        • If has connection (Ethernet/Wifi), continue.
        • If no connection, reconnect to network. Return to Step #4
      • Check EPG API connection
        • Open the Debug Console (press ‘d’ on the remote)
        • Check EPG API Latency
          • If PASS (has millisecond value), continue.
          • If FAIL, escalate/re-assign to Prasanna Gautam.
      • Check current Rundown state
        • Open the Debug Console (press ‘d’ on the remote)
        • Check “Current SC Rundowns” - current future, live, and past Rundown under EPG API
          • If elements exist, escalate/re-assign to DeRon Brown. Continue.
          • If no elements exist, escalate/re-assign to Prasanna Gautam.
        • Check current future, live, and past Rundown from the debug report.
          • If elements exist, escalate/re-assign to DeRon Brown.
          • If no elements exist, escalate/re-assign to Prasanna Gautam.

Live Rundown (Red) Element Failed to Tune

  1. Attempt to tune the live Rundown (red) element on a same setup stack. Follow the steps outlined in Step #4 then continue to Step #2.
  2. Contact user and arrange support on user’s device.
  3. Confirm proper deployment stack.
  4. Attempt to tune the live Rundown (red) element on the user’s stack.
    • If the element has the “tune in” option disabled, escalate/re-assign to DeRon Brown.
    • If it tunes you to SportsCenter:
      • If first time testing, resolve ticket as “Cannot Reproduce”.
      • Otherwise, resolve the ticket as “Fixed”.
    • If it fails to tune:
      • Check device network connection
        • If has connection (Ethernet/Wifi), continue.
        • If no connection, reconnect to network. Return to Step #4
      • Check EPG API connection
        • Open the Debug Console (press ‘d’ on the remote)
        • Check EPG API Latency
          • If PASS (has millisecond value), continue.
          • If FAIL, escalate/re-assign to Prasanna Gautam.
      • Check current Rundown state (if in an expected time)
        • Open the Debug Console (press ‘d’ on the remote)
        • Check current future, live, and past Rundown under EPG API
          • If elements exist, continue.
          • If no elements exist, escalate/re-assign to Prasanna Gautam.
        • Check current future, live, and past Rundown from the debug report
          • If elements exist, continue.
          • If no elements exist, escalate/re-assign to Prasanna Gautam.
      • Check device pairing
        • Open the Debug Console (press ‘d’ on the remote)
        • Check pairing device information:
          • If paired device is “corpmcast” or “ciscocatv”, check paired device ID:
            • If ID does not match deploy stack or is empty, follow the deployment instructions to pair the device. Return to Step #4
            • If ID matches, return to app and attempt to tune again:
              • If notification reads “failed to tune your device”, escalate/re-assign to Aakash Chokshi.
              • If notification reads “channel not available”, escalate/re-assign to Prasanna Gautam.
              • If you hear audio only, but no video, hard reset the Google TV (unplug) and return to Step #4.
              • Otherwise, escalate/re-assign to DeRon Brown.
          • If paired device is “googletv”, Test GTV tuning:
            • If information says “fail internal GTV error”, escalate/re-assign to DeRon Brown with a note to contact Google.
            • Otherwise, return to app and attempt to tune again:
              • If notification reads “failed to tune your device”, escalate/reassign to DeRon Brown.
              • If notification reads “channel not available”, escalate/reassign to Prasanna Gautam.
              • If you hear audio only, but no video, hard reset the Google TV (unplug) and return to Step #4.
              • Otherwise, escalate/reassign to DeRon Brown.
          • If paired device is “allhls”:
            • Check HLS server connection
              • Open the Debug Console (press ‘d’ on the remote)
              • Check HLS Video Server Latency
                • If PASS (has millisecond value), continue.
                • If FAIL, escalate/re assign to Matt Tretin.
            • Return to app and attempt to tune again:
              • If notification reads “channel not available”, escalate/re-assign to Prasanna Gautam.
              • If you hear audio only, but no video, hard reset the Google TV (unplug) and return to Step #4.
              • Otherwise, escalate/re-assign to DeRon Brown.

Hear Audio Only, But No Video

  1. Confirm proper deployment stack.
  2. Hard reset the Google TV (unplug).
  3. Check known HLS video
    • Open the Debug Console (press ‘d’ on the remote)
    • Under Video, play the test HLS video stream
      • If it plays correctly, close the ticket as “Fixed”.
      • If it fails to play, escalate/re-assign to DeRon Brown.

Not Receiving Likes or Shares

  1. Confirm proper deployment stack.
  2. Attempt to “Like” a rundown VOD:
    • If you see a “like” notifications
      • If first time testing, resolve the ticket as “Cannot Reproduce”
      • Otherwise, resolve the ticket as “Fixed”
    • If it fails:
      • Check device network connection
        • If has connection (Ethernet/Wifi), continue.
        • If no connection, reconnect to network. Return to Step #2
      • Check auto-standby settings
        • Press the Search button the keyboard side of the remote (magnifying glass)
        • Type “Settings” and select the Settings app
        • Go to “Unit Settings”
        • Check Auto-Standby
          • If it is “On”, change the setting to “Off”, restart the device, and return to Step #2
          • If it is “Off”, escalate/re-assign to DeRon Brown.
